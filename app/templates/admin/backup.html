1. Gerar backup

Produz um arquivo .zip com os dados de todas as tabelas em JSON e os arquivos enviados pelos orientandos. O formato é portátil: permanece restaurável caso o banco seja migrado para PostgreSQL.

O arquivo contém dados pessoais e hashes de senha. Guarde-o com o mesmo cuidado que daria ao próprio banco: não o deixe em pasta compartilhada nem o envie por canal aberto.

{{ gerar_form.hidden_tag() }} {{ gerar_form.submit() }}

2. Restaurar backup

Operação destrutiva. Todo o conteúdo atual — vínculos, marcos, documentos, atas, pareceres e a trilha de auditoria — é substituído pelo do arquivo enviado. Não há desfazer. Gere um backup do estado atual antes de prosseguir.

A restauração é recusada se o backup tiver sido gerado sob outra versão do esquema do banco, para não gravar dados sobre uma estrutura incompatível. {% if current_user.is_authenticated %}Se a sua conta não constar do arquivo, ela será preservada, de modo que você não fique sem acesso.{% endif %}

{{ render_form(restaurar_form, action=url_for('admin.restaurar_backup'), enctype='multipart/form-data') }}

Digite RESTAURAR no campo de confirmação. O tamanho do arquivo é limitado pela configuração de envio da aplicação (20 MB por padrão).

3. Apagar a base

Operação destrutiva e irreversível. Remove todos os usuários, vínculos, marcos, documentos, atas, pareceres, arquivos enviados e a trilha de auditoria. Preserva apenas a sua conta ({{ current_user.email }}), para que o sistema não fique sem administrador.

Um único registro novo de auditoria documenta o expurgo, com a data, o seu identificador e a quantidade removida de cada tipo. O histórico anterior não é recuperável — restaure um backup se precisar dele de volta.

{{ render_form(expurgar_form, action=url_for('admin.expurgar_base')) }}

Digite APAGAR no campo de confirmação.
